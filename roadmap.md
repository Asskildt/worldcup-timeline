## Next: app.js modularization

`app.js` is ~3000 lines. Natural split points:

```
filter.js   — ACTIVE_FILTER, setTeamFilter, getTeamBracketPaths
timeline.js — buildTimeline, renderTlToolbar
table.js    — buildTable, renderTblToolbar
bracket.js  — buildBracket, matchCard
modals.js   — openModal, openVenueModal, openTeamModal, openGroupModal
groups.js   — buildGroups, buildStats, buildArenas
ui.js       — updateCountdown, theme, CRT, buildNorwaySchedule
```

Requires either ES modules (import/export) or file concatenation in `build.js`. Best timing: after the tournament starts, during a natural feature pause.

## Multi-team filter

Let users select multiple teams in the filter simultaneously (max one group, but multiple individual teams).

- `ACTIVE_FILTER` changes from `{ type, value }` to `{ type: 'teams'|'group', values: Set<string> }`
- Click on a team in the filter list: toggle (add/remove from set), not replace
- Filter button shows a flag row for selected teams, with `+N` if many
- "Clear" removes all
- Group filter remains exclusive (one at a time)
- Special button: "Show favourites" — sets `values` to all favourite teams at once
- ~6–8 places in the code that check `=== ACTIVE_FILTER.value` must be updated to `.has()`

See `ideer.md` for full UX discussion.
